最近的感想

2020-12-01 11:57
Wang Ding

最近有在不断的写自己的感想，内容大多比较琐碎，但也感觉在逐渐摆脱琐碎和细节。

突然的感想就是：我并没有活在现在，而是活在了过去和未来。这从我平时记录的东西的内容和自己琐碎的动作就能看出来。

但是也有一些另外的感想：1. 很多突然来到的想法，会第一时间存到 gtd 中。其实这些突然来到的想法，大多是不成熟的想法。直觉它们是未来的映像，不需要立即就反应或马上开始想着完成。 2. 其实以上的想法，也大多是未完成的当下的主要的事情的一种反映而已。

linux 平台下 emacs native-comp 分支编译

2020-11-26 09:53
Wang Ding
  • 编译 libgccjit

    1. 下载 gcc

      git clone git://gcc.gnu.org/git/gcc.git

      然后切换到目录 cd gcc

    2. 切换分枝

      切换 gcc-9 分枝 git checkout remotes/origin/releases/gcc-9

      创建 build 目录 mkdir -p build

      进入 build 目录 cd build

    3. configure

      ../configure --enable-host-shared --enable-languages=jit,c,c++,objc,obj-c++ --enable-checking=release --disable-bootstrap

    4. 编译

      make -j8

    5. 安装（默认到 /usr/local ）

      sudo make install

    6. 加载库

      sudo ldconfig

      测试是否能发现 libgccjit 库 sudo ldconfig -p | grep libgccjit

  • 编译 emacs

    1. 下载 emacs

      克隆 emacs 仓库 git clone https://github.com/emacs-mirror/emacs

      进入 emacs 目录 cd emacs

      切换到 native-comp 分枝 git checkout feature/native-comp

    2. 进入 emacs 目录后运行 autogen.sh

      ./autogen.sh

    3. 编译 emacs

      ./configure --with-nativecomp CC=clang

    4. 安装（默认 /usr/local ）

      sudo make install

  • 启动 doom emacs

    1. 启动的时候如果报错

      comp-deferred-compilation-black-list

      然后找到这里解决：注释掉 .emacs.d/core/core.el 里面的一个过程 (add-to-list 'comp-deferred-compilation-black-list entry)

    2. doom build and run

      升级 doom ，包含 doom sync 和 doom build 过程 doom upgrade

      查看 doom 是否是使用 emacs 28 版本 doom run --version

      若确认是 emacs 28 版本，就可运行 doom run

  • 最后

    确实有流畅很多，也可能是心理作用吧。 :)

Difference Since Using Doom Emacs

2020-11-05 03:13
Wang Ding

自从使用了 doom emacs 后，就很少纠结系统升级了。以前没事总是会升级一下，但是现在最多的就是可能想起来会升级一下 doom emacs, 只需要在 emacs 中 upgrade 和 reload 即可。其实大多数时候我也只是 doom sync 一下配置然后 reload 一下，最多也就十几二十秒。

用 emacs 写东西简直是最近感觉最幸福的一件事情了。
