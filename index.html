Tutorial to workstation

2020-12-18 02:46
suzumiya natsuhi
  • SSH
    1. ssh 隧道内网穿透
      1. ssh 内网穿透

      2. autossh

        autossh 是对 ssh 的增量，因此使用 ssh 隧道来进行内网穿透，仅使用 autossh 即可

    2. 登录

      ssh user@hostname

    3. 无密码登录

      需要上传本地密钥到远程主机用户的 ~/.ssh/known_hosts 文件中

      1. 生成本地密钥 ssh-keygen
      2. 上传本地密钥 ssh-copy-id
    4. 更快登录
      1. 设置 .ssh/config
  • 文件传输

    1. 采用 Filezilla 传输文件

      最方便的一种方式，根据 ssh 信息来配置即可

    2. 采用 scp 传输少量文件

      在实现 ssh 免密码的基础上，在命令行 scp [filename] [remote-host] 即可

    3. 采用 git 同步文件

      需要 git 操作基础

  • 运行代码

    1. python 运行脚本

      python test.py

    2. 后台运行

      后台运行在命令后面加入 & ，也即命令成为 python test.py & 即可。

    3. 存储日志

      在代码中，在关键信息的地方打印日志即可。

    4. 存储图像

      代码在后台运行时，无需使用 matplotlib.show() 来查看图像，因此保存 matlplotlib.savefig(filename=filename) 即可。

  • 打印关键信息和保存日志以及查看日志
    1. 打印日志
      1. 代码中添加需要打印的信息
    2. 保存日志

      1. 使用文件重定向

        例如 python test.py > log.txt

      2. 同时将错误信息重定向

        2>&1

    3. 查看日志
      1. 使用 Pycharm 设置 sftp 来查看文件，最方便
      2. ssh 登录远程机器，然后使用 more or less 命令查看
      3. Emacs 的 tramp, 次方便。

Emacs 下同时查看和编辑一个文件的多个 headline: 使用 Indirect Buffer

2020-12-15 08:04
suzumiya natsuhi

在使用 emacs 编辑一个很多的 headlines 的 org 文件时，如果每个 headline 下的内容需要交叉查看和编辑的时候，我就非常需要一个单独的 windows(emacs 下的窗口，下均同) 仅显示一个 headline 下的所有内容，所以必须用到 org-narrow-to-subtree 这个函数。

但是在第一次使用这个函数后，就无法看到其他的 headlines ，所以想在多个窗口中打开多个 headlines 仅使用 org-narrow-to-subtree 是不够的。好在有了 clone-indirect-buffer 可以复制一个当前文件的 buffer 。一般情况下，在使用 emacs 打开文件的时候，我们只能在将一个文件绑定在一个 buffer 中，这个函数可以让我们将一个文件绑定在多个 buffer 中。

然后在不同的 windows 中打开相同文件的不同 buffer ，可以看到这个文件的不同 headlines ，进而使用对不同的 headline 使用 org-narrow-to-subtree 来实现“在不同的 windows 里面查看一个文件中不同 headlines 的 org-narrow-subtree ”。

这个功能对于我来说很实用，尤其是对一个比较长的， subtree 又比较多的 org 文件中同时查看和编辑多个 headline 的情况下尤其有用。
