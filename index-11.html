Open-close principle

2021-01-17 11:33
Wang Ding

开闭原则是软件工程中对于开发原则的一条描述。它规定：软件中的对象（类，模块，函数等等）应该对于拓展开放，对于修改封闭。这是一条非常重要的原则，它的内涵是：一个实体是允许在不改变它本身（的源代码）的前提下变更它的行为。

这段话我所理解的是：只加不改。也就是对于一个能够运行正常的代码（或者说旧的系统），我们最好是添加一些需要的东西，而不是修改内部的机制。因为内部的机制牵连比较多，修改的时候也比较容易出现引用的问题。但是增加就不用想那么多，也实际上效果更好一点。

对于旧的系统，我的一个理解也包含生活方式或者理念的系统。在接触到一个新的生活方式或者理念的时候，不要急着先否定过去的系统，而是加上去新的机制和理念。它们不应该是互斥存在的，而是可以友好的协作的。如果不行，那么只能说没有组织好。

很有意思的一条原则，发散的话可以得到很多有价值的应用。

Turn to archlinux

2021-01-04 11:35
Wang Ding

昨天晚上新装了 archlinux ，目前基本已经能用了。

其实并没有太多理由非要用 archlinux 。 openSUSE 用得还行，唯二让我换的理由是：1. fcitx-rime 出了一点问题，用了一段时间后，虽然切换到中文，但是无法打出中文。 2. 装各种包有点麻烦。这点 archlinux 可以说完全解决痛点。

早就想换，昨天晚上干脆就下定决心借来了 U 盘开始装 archlinux 。全程也没有花费太多时间（也就十多个小时吧。

装好了之后，装上了 xfce 桌面，emacs 启动速度也很快，用起来两个字：开心。

虽然还有一些配置文件还没有搬运过来，直接挂载上了 openSUSE 的之前 home 目录，后面想到了就再配置。

Emacs 自动保存文件设置

2020-12-23 12:06
Wang Ding

Emacs 自带文件自动保存的机制，利用 do-auto-save 可以自动存储当前编辑的文件为一个编辑过的复本，并未直接存储到 buffer 对应的文件中。当然也可以直接开启全局的 (auto-save-visited-mode t) ，在默认的 auto-save-visited-interval 时间（默认 5 秒），后会自动将已经修改后的 buffer 内容写入文件中。

其中使用 do-auto-save 保存的文件的复本可以在打开该文件本身时有提示。这是一个好的地方，也是容易忽略的地方：某次并没有将改动保存到文件，而是自动保存到了某个复本，而下一次用 emacs 打开该文件，虽然 emacs 有提醒该文件有复本未保存到文件中，此时极有可能形成了两个新的基于原文件改动的分枝。众所周知，合并文件是一件耗时的事情。所以我觉得还是打开 (auto-save-visited-mode t) 更好，避免形成两个改动文件并且需要合并内容的情形。

Tutorial to workstation

2020-12-18 02:46
Wang Ding

  • 代理

    目前最爽的解决方案是透明代理。

  • SSHD

    开启服务器的 sshd 服务

  • SSH
    1. ssh 隧道内网穿透
      1. ssh 内网穿透
        • 实施条件
          • 公共 IP 主机（VPS），也称“跳板机”
          • 需要访问的内网机器（待穿透的内网机器）能访问并 SSH 登录公共主机
          • 本地机器（访问内网机器的机器）能访问并 SSH 登录中间主机
        • 实施过程

          • 内网穿透到跳板机 

                        ssh -qTfNn -R '[::]:22222:localhost:22' [Jump Server User]@[Jumper Server IP or Domain Name]

          • 本地机器访问内网机器 

                        ssh -p 22222 [Remote User on machine on sub-network]@[Jump Server IP or Domain Name]

      2. autossh

        autossh 是对 ssh 的增强，因此使用 ssh 隧道来进行内网穿透，仅使用 autossh 即可

    2. 登录

      ssh user@hostname

    3. 无密码登录

      需要上传本地密钥到远程主机用户的 ~/.ssh/known_hosts 文件中

      1. 生成本地密钥 ssh-keygen
      2. 上传本地密钥 ssh-copy-id
    4. 更快登录
      1. 设置 .ssh/config
  • 文件传输

    1. 采用 Filezilla 传输文件

      最方便的一种方式，根据 ssh 信息来配置即可

    2. 采用 scp 传输少量文件

      在实现 ssh 免密码的基础上，在命令行 scp [filename] [remote-host] 即可

    3. 采用 git 同步文件

      需要 git 操作基础

  • 运行代码

    1. python 运行脚本

      python test.py

      1. 脚本参数
        1. 参考 python 参数使用

      2. 脚本循环

        有时需要跑一个耗时较长的程序，写一个脚本在 python 运行时传入参数来自动化程序运行 

        
#!/bin/bash
# cmd='nohup python -u ga_kpca.py > "log/ga-custom-kernel-kpca_"$(date "+%Y-%m-%d_%H-%M-%S").log 2>&1'

cd ~/wangding/ga-kpca
conda activate "wangding"

for i in $(seq 2 7)
do
    for j in $(seq 2 7)
    do
        nohup python -u ga_kpca.py $i $j > "log/ga-custom-kernel-kpca_"$(date "+%Y-%m-%d_%H-%M-%S").log 2>&1
    done
done

        其中 $i 和 $j 就是在 bash 中传入 python 的参数，参数可以结合具体代码逻辑做任何事情。

    2. 后台运行

      后台运行在命令后面加入 & ，也即命令成为 python test.py & 即可。

    3. 存储日志

      在代码中，在关键信息的地方打印日志即可。

    4. 存储图像

      代码在后台运行时，无需使用 matplotlib.show() 来查看图像，因此保存 matlplotlib.savefig(filename=filename) 即可。

  • 打印关键信息和保存日志以及查看日志
    1. 打印日志
      1. 代码中添加需要打印的信息
    2. 保存日志

      1. 使用文件重定向

        例如 python test.py > log.txt

      2. 同时将错误信息重定向

        2>&1

    3. 查看日志
      1. 使用 Pycharm 设置 sftp 来查看文件，最方便
      2. ssh 登录远程机器，然后使用 more or less 命令查看
      3. Emacs 的 tramp, 次方便。
